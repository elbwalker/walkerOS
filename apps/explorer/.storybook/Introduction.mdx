# walkerOS Explorer Component Library

Interactive React components for walkerOS documentation and exploration.

## Overview

The `@walkeros/explorer` package provides a professional component library for building interactive walkerOS documentation, demos, and configuration tools.

## Architecture

Components follow **atomic design principles**:

- **Atoms**: Basic UI elements (buttons, inputs, toggles)
- **Molecules**: Component combinations (code editors, navigation)
- **Organisms**: Complex integrated systems (full editors, config UI)
- **Demos**: Complete workflows (mapping demos, destination testing)

## Installation

```bash
npm install @walkeros/explorer
```

## Usage

Import components and styles:

```tsx
import { CodeBox } from '@walkeros/explorer';
import '@walkeros/explorer/styles.css';

function App() {
  return (
    <CodeBox
      code="const event = { entity: 'product', action: 'view' };"
      language="javascript"
      label="Event Code"
      showCopy
    />
  );
}
```

## Theme Support

Components support light and dark themes via `data-theme` attribute:

```html
<div data-theme="light">
  <!-- Components render with light theme -->
</div>

<div data-theme="dark">
  <!-- Components render with dark theme -->
</div>
```

Use the theme toolbar in Storybook to switch between light and dark modes.

## Key Features

### Monaco Editor Integration
- Full TypeScript/JavaScript syntax highlighting
- Auto-completion with walkerOS type definitions
- Multiple language support (JSON, HTML, CSS, etc.)

### RJSF Forms
- React JSON Schema Forms integration
- Custom field and widget components
- Validation and error handling

### Mapping System
- Visual mapping configuration editor
- Tree navigation for complex configs
- Live preview and transformation

### Destination Testing
- Real-time destination integration testing
- Event processing visualization
- Configuration playground

## walkerOS Integration

Demo components require walkerOS packages as peer dependencies:

- `@walkeros/core` - **Required** for all components
- `@walkeros/collector` - Optional, for demo components
- `@walkeros/web-source-browser` - Optional, for browser demos

4. Document props with JSDoc comments
5. Test in both light and dark themes
