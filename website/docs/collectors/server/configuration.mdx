There are a few configuration options when creating a server collector instance:

```ts
const { instance } = createServerCollector({
  consent: {
    functional: true,
  },
  settings: {
    foo: 'bar',
  },
  destinations: {
    log: {
      type: 'log',
      push: console.log,
    },
  },
  globalsStatic: {
    version: '3.1.4',
  },
  hooks: {
    prePush: (params, event) => {
      console.log('prePush', event.event);
      return params.fn(event);
    },
  },
  on: {
    run: [(instance) => console.log('run', instance.round)],
  },
  onError: (error) => {
    console.error('server collector', error);
  },
  onLog: (message) => {
    console.log('server source', message);
  },
  tagging: 1,
  user: {
    id: 'user123',
  },
  verbose: true,
});
```

| Option        | Type     | Description                                         |
| ------------- | -------- | --------------------------------------------------- |
| consent       | object   | Initial consent states                              |
| contracts     | object   | Contract schemas for event validation               |
| custom        | object   | Individual custom data                              |
| destinations  | object   | Destinations that might process the events          |
| globalsStatic | object   | Static values for the globals that persist new runs |
| hooks         | object   | Functions to customize default behavior             |
| on            | object   | Event listeners for specific walkerOS events        |
| onError       | function | Custom error handler                                |
| onLog         | function | Custom logging for the verbose mode                 |
| tagging       | number   | Helpful to differentiate the used setup version     |
| user          | object   | Handles the user IDs                                |
| verbose       | boolean  | To enable verbose mode                              |
