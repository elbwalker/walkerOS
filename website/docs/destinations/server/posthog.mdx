# PostHog

<PackageLink env="server" github="packages/server/destinations/posthog" npm="@walkeros/server-destination-posthog" />

Server-side event delivery to PostHog for product analytics, identity resolution, group analytics, and feature flags via the official `posthog-node` SDK.

:::info Where This Fits
PostHog is a **server destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'PostHog', highlight: true } }}
/>

Sends events server-side to PostHog's ingestion API, translating walkerOS events into `capture`, `identify`, and `groupIdentify` calls with full control over identity and group resolution.
:::

## Docker Deployment

### Step 1: Create Flow Configuration

Use the `$env.NAME` syntax in your flow configuration to reference environment variables at runtime:

<CodeSnippet
  code={`{
  "destinations": {
    "posthog": {
      "package": "@walkeros/server-destination-posthog",
      "config": {
        "settings": {
          "apiKey": "$env.POSTHOG_API_KEY",
          "host": "$env.POSTHOG_HOST:"
        }
      }
    }
  }
}`}
  language="json"
  title="flow.json"
/>

### Step 2: Bundle the Flow

Bundle your flow configuration before running in Docker:

<CodeSnippet
  code={`npx walkeros bundle flow.json -o dist/bundle.mjs`}
  language="bash"
/>

### Step 3: Run with Docker

<CodeSnippet
  code={`docker run -d \\
  -e POSTHOG_API_KEY=phc_your-project-api-key \\
  -e POSTHOG_HOST=https://eu.i.posthog.com \\
  -v ./dist/bundle.mjs:/app/flow/bundle.mjs:ro \\
  -p 8080:8080 \\
  walkeros/flow`}
  language="bash"
/>

### Docker Compose Example

<CodeSnippet
  code={`services:
  walkeros:
    image: walkeros/flow:latest
    environment:
      - POSTHOG_API_KEY=\${POSTHOG_API_KEY}
      - POSTHOG_HOST=\${POSTHOG_HOST}
    volumes:
      - ./dist/bundle.mjs:/app/flow/bundle.mjs:ro
    ports:
      - "8080:8080"`}
  language="yaml"
/>

:::tip Security Best Practice
Never commit API keys to version control. Use environment variables or a secrets manager to inject credentials at runtime.
:::
