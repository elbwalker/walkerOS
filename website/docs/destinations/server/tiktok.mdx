# TikTok Events API

<PackageLink env="server" github="packages/server/destinations/tiktok" npm="@walkeros/server-destination-tiktok" />

Server-side event delivery to TikTok's Events API for enhanced conversion tracking, bypassing browser limitations and improving match quality for ad optimization.

:::info Where This Fits
TikTok Events API is a **server destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'TikTok Events API', highlight: true } }}
/>

Sends events server-side to TikTok's Events API using raw HTTP (no SDK). Identity fields (email, phone_number, external_id) are SHA-256 hashed before sending. Events include a stable `event_id` for deduplication with the browser pixel.
:::

## Hashing

The destination automatically SHA-256 hashes `email`, `phone_number`, and `external_id` before sending to TikTok. Pass raw values and the hashing is handled for you. If your data is already hashed or you want to skip hashing for specific fields, use the `doNotHash` setting:

<CodeSnippet
  code={`"settings": {
  "accessToken": "YOUR_ACCESS_TOKEN",
  "pixelCode": "YOUR_PIXEL_CODE",
  "doNotHash": ["email"]
}`}
  language="json"
/>

## Deduplication

Each event is sent with `event_id` set to the walkerOS event `id`. If you also run the [TikTok browser pixel](/docs/destinations/web/tiktok), TikTok deduplicates server and browser events that share the same `event_id`, so conversions are not double-counted.

## Docker deployment

### Step 1: Create flow configuration

Use the `$env.NAME` syntax in your flow configuration to reference environment variables at runtime:

<CodeSnippet
  code={`{
  "destinations": {
    "tiktok": {
      "package": "@walkeros/server-destination-tiktok",
      "config": {
        "settings": {
          "accessToken": "$env.TIKTOK_ACCESS_TOKEN",
          "pixelCode": "$env.TIKTOK_PIXEL_CODE",
          "test_event_code": "$env.TIKTOK_TEST_CODE:"
        }
      }
    }
  }
}`}
  language="json"
  title="flow.json"
/>

### Step 2: Bundle the flow

Bundle your flow configuration before running in Docker:

<CodeSnippet
  code={`npx walkeros bundle flow.json -o dist/bundle.mjs`}
  language="bash"
/>

### Step 3: Run with Docker

<CodeSnippet
  code={`docker run -d \\
  -e TIKTOK_ACCESS_TOKEN=your-access-token \\
  -e TIKTOK_PIXEL_CODE=YOUR_PIXEL_CODE \\
  -v ./dist/bundle.mjs:/app/flow/bundle.mjs:ro \\
  -p 8080:8080 \\
  walkeros/flow`}
  language="bash"
/>

### Docker Compose example

<CodeSnippet
  code={`services:
  walkeros:
    image: walkeros/flow:latest
    environment:
      - TIKTOK_ACCESS_TOKEN=\${TIKTOK_ACCESS_TOKEN}
      - TIKTOK_PIXEL_CODE=\${TIKTOK_PIXEL_CODE}
    volumes:
      - ./dist/bundle.mjs:/app/flow/bundle.mjs:ro
    ports:
      - "8080:8080"`}
  language="yaml"
/>

:::tip Security best practice
Never commit access tokens to version control. Use environment variables or a secrets manager to inject credentials at runtime.
:::
