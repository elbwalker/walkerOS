# Amplitude

<PackageLink
  env="web"
  github="packages/web/destinations/amplitude"
  npm="@walkeros/web-destination-amplitude"
/>

[Amplitude](https://amplitude.com/) is a product analytics platform with
built-in identity, revenue, groups, and optional session replay, feature
experiments, and guides & surveys. This destination forwards walkerOS events
to Amplitude via the official
[`@amplitude/unified`](https://www.npmjs.com/package/@amplitude/unified)
SDK, translating events into `amplitude.track(...)`, `identify(...)`,
`revenue(...)`, `setGroup(...)`, and `setOptOut(...)` calls.

:::info Where This Fits
Amplitude is a **web destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'Amplitude', highlight: true } }}
/>

Loads the Amplitude Analytics SDK, forwards every walkerOS event, and lets
you layer on identity, revenue, groups, consent toggling, and optional
plugins.
:::

## Consent

Declare the required consent keys on `config.consent`. When a walker
consent event arrives, the destination's `on('consent')` handler toggles
`amplitude.setOptOut()` with strict semantics: all required keys must be
granted for opt-in, any missing grant opts out.

```typescript
config: {
  consent: { analytics: true },
  settings: { apiKey: 'YOUR_KEY' },
}
```

## Plugins

All three plugin packages are npm-bundled with the destination (~135 KB
for Session Replay plus Experiment and Engagement payloads). Enable them
via destination settings:

```typescript
settings: {
  apiKey: 'YOUR_KEY',
  sessionReplay: { sampleRate: 1 },
  experiment: { deploymentKey: 'DEPLOYMENT_KEY' },
  engagement: true,
}
```
