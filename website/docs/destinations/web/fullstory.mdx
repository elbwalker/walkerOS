# FullStory

<PackageLink
  env="web"
  github="packages/web/destinations/fullstory"
  npm="@walkeros/web-destination-fullstory"
  status="beta"
/>

[FullStory](https://www.fullstory.com/) delivers enterprise session replay and
digital-experience analytics. This destination forwards walkerOS events to
FullStory via the official
[`@fullstory/browser`](https://www.npmjs.com/package/@fullstory/browser) SDK
v2, calling `FullStory('trackEvent', ...)` for events and supporting
identity, user/page properties, and consent gating via `start`/`shutdown`.

:::info Where This Fits
FullStory is a **web destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'FullStory', highlight: true } }}
/>

Loads the FullStory SDK, forwards every walkerOS event as
`FullStory('trackEvent', { name, properties })`, and lets you layer on
identity, user/page properties, and consent.
:::

## Mapping examples

:::tip

The code examples are interactive. You can edit the code to generate your
individual event mapping.

:::

### Default event forwarding

Every walkerOS event becomes `FullStory('trackEvent', { name, properties })`
without any mapping.

<DestinationDemo
  destination={destination}
  settings={{ orgId: 'o-XXXXXX-na1' }}
  event={data.examples.step.defaultEventForwarding.in}
  mapping={data.examples.step.defaultEventForwarding.mapping}
  generic={true}
/>

### Rename the forwarded event

Use `mapping.name` to emit a different name to FullStory.

<DestinationDemo
  destination={destination}
  settings={{ orgId: 'o-XXXXXX-na1' }}
  event={data.examples.step.mappedEventName.in}
  mapping={data.examples.step.mappedEventName.mapping}
  generic={true}
/>

### Identify

Resolve a mapping value into `{ uid, properties }` and call
`FullStory('setIdentity', ...)` before the event.

<DestinationDemo
  destination={destination}

Persist user-level properties across sessions via `FullStory('setProperties', { type: 'user', ... })`.

## Consent

FullStory supports runtime consent via `start`/`shutdown` (recording on/off) or
the `consent` flag on `setIdentity`. Map walkerOS consent keys to FullStory
actions via `settings.consent`, and combine with `startCaptureManually: true`
to delay recording until consent is granted:

<CodeSnippet
  code={`destinations: {
  fullstory: {
    code: destinationFullStory,
    config: {
      consent: { analytics: true },
      settings: {
        orgId: 'o-XXXXXX-na1',
        startCaptureManually: true,
        consent: {
          analytics: 'capture',
        },
      },
    },
  },
}`}
  language="typescript"
/>

- `"capture"` toggles `start`/`shutdown` (recording on/off)
- `"consent"` sets the `consent` flag on `setIdentity`
