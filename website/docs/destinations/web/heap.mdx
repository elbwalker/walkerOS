# Heap

<PackageLink
  env="web"
  github="packages/web/destinations/heap"
  npm="@walkeros/web-destination-heap"
  status="beta"
/>

[Heap](https://www.heap.io/) (now part of Contentsquare) is a product
analytics platform with auto-capture and retroactive event definitions.
This destination loads the Heap snippet from Heap's CDN and forwards
walkerOS events through `heap.track(name, properties)`, plus identity,
user properties, persistent event properties, and runtime consent.

:::info Where This Fits
Heap is a **web destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'Heap', highlight: true } }}
/>

Loads the Heap snippet, forwards every walkerOS event as
`heap.track(name, properties)`, and lets you layer on identity, user
properties, and consent translation.
:::

## Examples

:::tip

The examples below show input, mapping, and the resulting vendor calls.

:::

### Default event forwarding

Every walkerOS event becomes `heap.track(name, {})` without any mapping.

<StepExample example={data.examples.step.defaultEventForwarding} />

### Rename + map event properties

Use `mapping.name` to emit a different name and `mapping.data.map` to
shape the properties argument.

<StepExample example={data.examples.step.destinationLevelInclude} />

### Identify on login

Resolve a mapping value into a string identity and call `heap.identify(id)`
with user properties. `skip: true` suppresses the default track call.

<StepExample example={data.examples.step.userLoginIdentify} />

### Reset on logout

Truthy `mapping.settings.reset` triggers `heap.resetIdentity()`.

<StepExample example={data.examples.step.userLogoutReset} />

### Persistent event properties

`heap.addEventProperties(...)` stores properties on every subsequent
event until cleared. Combine with `skip: true` when the event itself is
only a trigger for the property update.

<StepExample example={data.examples.step.globalEventProperties} />

## Consent

The destination wires walkerOS `config.consent` to Heap's runtime consent
API. `heap.startTracking()` fires when all required consent keys are
granted; `heap.stopTracking()` fires otherwise.

<CodeSnippet
  code={`destinations: {
  heap: {
    code: destinationHeap,
    config: {
      consent: { analytics: true },
      settings: { appId: 'YOUR_HEAP_APP_ID' },
    },
  },
}`}
  language="typescript"
/>
