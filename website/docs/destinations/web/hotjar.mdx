# Hotjar

<PackageLink
  env="web"
  github="packages/web/destinations/hotjar"
  npm="@walkeros/web-destination-hotjar"
  status="beta"
/>

[Hotjar](https://www.hotjar.com/) provides session recordings, heatmaps, and
on-page feedback for web products. This destination forwards walkerOS events to
Hotjar via the official
[`@hotjar/browser`](https://www.npmjs.com/package/@hotjar/browser) SDK,
translating events into `Hotjar.event(...)` calls and handling identity and
SPA state changes.

:::info Where This Fits
Hotjar is a **web destination** in the walkerOS flow:

<FlowMap
  sources={{ default: { highlight: false } }}
  collector={{ highlight: false }}
  destinations={{ default: { label: 'Destination', text: 'Hotjar', highlight: true } }}
/>

Loads the Hotjar SDK, forwards every walkerOS event as `Hotjar.event(name)`,
and lets you layer on identity and SPA route tracking.
:::

## Examples

:::tip

The examples below show input, mapping, and the resulting vendor calls.

:::

### Default event forwarding

Every walkerOS event becomes `Hotjar.event(name)` without any mapping.

<StepExample example={data.examples.step.defaultEventForwarding} />

### Rename the forwarded event

Use `mapping.name` to emit a different name to Hotjar.

<StepExample example={data.examples.step.renamedEvent} />

### Identify

Resolve a mapping value into `{ userId, ...attributes }` and call
`Hotjar.identify(userId, attributes)` before the event.

<StepExample example={data.examples.step.userLoginIdentify} />

### SPA state change

For single-page applications, emit `Hotjar.stateChange(path)` so heatmaps
aggregate by virtual URL. Combine with `skip: true` to suppress the default
`Hotjar.event(...)` call.

<StepExample example={data.examples.step.pageViewStateChange} />

## Consent

Hotjar has no runtime consent API. Consent is gated at the walkerOS level via
`config.consent`:

<CodeSnippet
  code={`destinations: {
  hotjar: {
    code: destinationHotjar,
    config: {
      consent: { marketing: true },
      settings: { siteId: 1234567 },
    },
  },
}`}
  language="typescript"
/>

Unconsented events are blocked by the collector before reaching the
destination.
