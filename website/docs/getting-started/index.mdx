# Installation

Choose the installation method that best fits your project and technical
requirements.

## Quick Start

### 30-Second Setup

<Tabs>
  <TabItem value="npm" label="NPM" default>

For projects using npm and build tools:

```bash
npm install @walkerOS/collector
```

```javascript
import { createCollector } from '@walkerOS/collector';

const { collector, elb } = await createCollector({
  destinations: {
    console: { push: console.log },
  },
});

await elb('page view', { title: 'Homepage' });
```

  </TabItem>
  <TabItem value="browser" label="Browser/CDN">

For websites without build tools:

```html
<!-- Add to your HTML head -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/walker.js@latest/dist/walker.js"></script>

<script>
  // Configure tracking
  window.elbConfig = {
    collector: {
      destinations: {
        console: { push: (event) => console.log('Event:', event) },
      },
    },
  };
</script>
```

  </TabItem>
</Tabs>

## Choose Your Method

### üîß NPM Installation

**Best for**: Developers using modern build tools (Webpack, Vite, etc.)

**Advantages**:

- Type safety with TypeScript
- Tree shaking and optimization
- Version control and dependency management
- Hot module replacement during development

**Use when**:

- Building React, Vue, or Angular applications
- Using bundlers like Webpack, Vite, or Rollup
- Working in a team development environment

[‚Üí See NPM Installation Guide](#npm-installation)

### üåê Browser/CDN Installation

**Best for**: Marketing teams, agencies, legacy websites

**Advantages**:

- No build tools required
- Copy-paste implementation
- Works with any HTML page
- Instant setup and testing

**Use when**:

- Adding tracking to existing websites
- Working with content management systems
- Quick prototyping or testing
- Limited technical resources

[‚Üí See Browser Installation Guide](/docs/sources/web/browser/)

### üì¶ Walker.js Bundle

**Best for**: Complete solution without configuration

**Advantages**:

- Everything included in one file
- Automatic DOM tracking
- DataLayer compatibility
- No package management needed

**Use when**:

- Migrating from Google Tag Manager
- Need comprehensive tracking quickly
- Working with multiple websites
- Want automatic event collection

[‚Üí See Walker.js Documentation](/docs/apps/walkerjs)

## NPM Installation

### Core Packages

#### Basic Setup

For event collection and processing:

```bash
npm install @walkerOS/collector
```

#### Web Development

For browser-based applications:

```bash
# Basic web tracking
npm install @walkerOS/collector @walkerOS/web-source-browser

# With Google Analytics
npm install @walkerOS/collector @walkerOS/web-destination-gtag

# Complete web setup
npm install @walkerOS/collector @walkerOS/web-source-browser @walkerOS/web-destination-gtag @walkerOS/web-destination-meta
```

#### Server Development

For Node.js and server environments:

```bash
# Basic server tracking
npm install @walkerOS/collector @walkerOS/server-destination-aws

# Complete server setup
npm install @walkerOS/collector @walkerOS/server-destination-aws @walkerOS/server-destination-gcp
```

### Package Categories

#### Sources (Data Collection)

```bash
# Browser DOM tracking
npm install @walkerOS/web-source-browser

# DataLayer integration
npm install @walkerOS/web-source-datalayer
```

#### Web Destinations (Analytics Platforms)

```bash
# Google Analytics 4 & Ads
npm install @walkerOS/web-destination-gtag

# Meta Pixel (Facebook/Instagram)
npm install @walkerOS/web-destination-meta

# PiwikPro Analytics
npm install @walkerOS/web-destination-piwikpro

# Plausible Analytics
npm install @walkerOS/web-destination-plausible

# Custom API endpoints
npm install @walkerOS/web-destination-api
```

#### Server Destinations (Data Streaming)

```bash
# AWS Kinesis Data Firehose
npm install @walkerOS/server-destination-aws

# Google Cloud Pub/Sub
npm install @walkerOS/server-destination-gcp

# Meta Conversions API
npm install @walkerOS/server-destination-meta
```

### TypeScript Support

All packages include TypeScript definitions:

```typescript
import { createCollector } from '@walkerOS/collector';
import { destinationGtag } from '@walkerOS/web-destination-gtag';
import { sourceBrowser } from '@walkerOS/web-source-browser';
import type { WalkerOS, Collector } from '@walkerOS/core';

const { elb }: { elb: WalkerOS.Elb } = await createCollector({
  sources: {
    browser: { code: sourceBrowser },
  },
});

// Add destination with configuration
elb('walker destination', destinationGtag, {
  settings: {
    ga4: { measurementId: 'G-XXXXXXXXXX' },
  },
});
```

## Browser/CDN Installation

### Individual Packages

Load packages directly from CDN:

```html
<!-- Core collector -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/collector/dist/index.browser.js"></script>

<!-- Browser source -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/web-source-browser/dist/index.browser.js"></script>

<!-- Google Analytics destination -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/web-destination-gtag/dist/index.browser.js"></script>

<script>
  // Use the loaded packages
  const { createCollector } = walkerOSCollector;
  const { sourceBrowser } = walkerOSWebSourceBrowser;
  const { destinationGtag } = walkerOSWebDestinationGtag;
</script>
```

### ES6 Modules

Use modern ES6 modules:

```html
<script type="module">
  // Dynamic imports
  const { createCollector } = await import(
    'https://cdn.jsdelivr.net/npm/@walkerOS/collector/dist/index.mjs'
  );
  const { destinationGtag } = await import(
    'https://cdn.jsdelivr.net/npm/@walkerOS/web-destination-gtag/dist/index.mjs'
  );
  const { sourceBrowser } = await import(
    'https://cdn.jsdelivr.net/npm/@walkerOS/web-source-browser/dist/index.mjs'
  );

  // Setup tracking
  const { elb } = await createCollector({
    sources: {
      browser: { code: sourceBrowser },
    },
  });

  // Add destination with configuration
  elb('walker destination', destinationGtag, {
    settings: {
      ga4: { measurementId: 'G-XXXXXXXXXX' },
    },
  });

  // Track events
  await elb('page view', { title: document.title });
</script>
```

### Legacy Browser Support

For older browsers, use ES5 builds:

```html
<!-- ES5 compatible builds -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/collector/dist/index.es5.js"></script>
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/web-destination-gtag/dist/index.es5.js"></script>
```

## Advanced Options

### Version Pinning

Pin to specific versions for production:

```html
<!-- Pin to specific version -->
<script src="https://cdn.jsdelivr.net/npm/@walkerOS/walker.js@0.0.7/dist/walker.js"></script>
```

```bash
# NPM version pinning
npm install @walkerOS/collector@0.0.7
```

### Self-Hosting

Download and host files yourself:

```bash
# Download specific package
npm pack @walkerOS/walker.js
tar -xzf walkerOS-walker.js-*.tgz
cp package/dist/* ./public/js/
```

```html
<!-- Use self-hosted files -->
<script src="/js/walker.js"></script>
```

### Build Variants

Different builds for different needs:

| Build Type    | Use Case               | Example            |
| ------------- | ---------------------- | ------------------ |
| `.browser.js` | Modern browsers (IIFE) | `index.browser.js` |
| `.es5.js`     | Legacy browsers (ES5)  | `index.es5.js`     |
| `.mjs`        | ES6 modules            | `index.mjs`        |
| `.js`         | CommonJS/Node.js       | `index.js`         |
