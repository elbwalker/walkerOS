# Validator

<PackageLink
  env="server"
  github="packages/transformers/validator"
  npm="@walkeros/transformer-validator"
/>

:::warning Deprecated for direct use
Prefer the step-level [`validate:`](/docs/getting-started/flow/validate) primitive. The walkerOS CLI auto-injects this transformer when `validate:` is declared on any step. This page documents the underlying transformer for users who need explicit chain wiring (for example a catch-all validator upstream of every destination).
:::

The validator runs events through JSON Schema using AJV. Invalid events are dropped and logged, valid events pass through unchanged. The recommended way to declare validation is the step-level `validate:` primitive, which auto-injects this transformer at the right chain position. Use the transformer directly only when you need explicit placement, for example one validator sitting in front of every destination via `before`.

## Explicit wiring

Place the validator at the head of every destination's `before` chain to gate the whole fan-out behind a single contract.

<CodeSnippet
  code={`"flows": {
  "default": {
    "transformers": {
      "validate": {
        "package": "@walkeros/transformer-validator",
        "config": {
          "settings": {
            "format": true,
            "events": {
              "order": {
                "complete": {
                  "properties": { "data": { "required": ["total"] } }
                }
              }
            }
          }
        }
      }
    },
    "destinations": {
      "ga4": {
        "package": "@walkeros/web-destination-gtag-ga4",
        "before": ["validate"]
      },
      "meta": {
        "package": "@walkeros/web-destination-meta-pixel",
        "before": ["validate"]
      }
    }
  }
}`}
  language="json"
/>

A failed check drops the event globally (it never reaches any step that lists `validate` in its `before`). This is the difference from step-level `validate:`, which skips only the step it is attached to.

## Settings

| Field | Type | Description |
|-------|------|-------------|
| `format?` | `boolean` | Check the full `WalkerOS.Event` structural shape (name, source, timestamp, etc.). Defaults to `true`. |
| `events?` | `Record<entity, Record<action, JsonSchema>>` | Per-event JSON Schemas keyed by entity then action, with `*` wildcard fallback. |
| `schema?` | `JsonSchema` | A single JSON Schema applied to every event that reaches the transformer. |

`events` looks up rules by `entity.action` with fallback through `entity.*`, `*.action`, `*.*`, first match wins. Events with no matching entry pass the events layer unchanged.
