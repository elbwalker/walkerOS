These helper utils are a collection of useful functions used by walkerOS. They
can be used as standardtized building blocks for custom setups.

Learn how to install and use them in the <Link to="./#installation">
Utils/Installation </Link> section.

## getByStringDot

Access values from an event by using the string-dot notation. Use the key
`"data.id"` on `event: { data: { id: "wow" } }` to get the value `wow`.

```js
// getByStringDot(event, key, i=0): unknown
getByStringDot({ data: { id: 'wow' } }, 'data.id');

// returns "wow"
```

Use the wildcard character `_` and `i` for dynamic access, like `"nested._.id"`
on `{ nested: [ 1, 2, { id: "impressive" } ] }` to get `impressive`.

If the key doesn't exist the the function returns `undefined`.

## getMarketingParameters

```js
// getMarketingParameters(url: string, custom={}): WalkerOS.Properties
getMarketingParameters("https://localhost/?utm_source=docs&gclid=xxx");

// returns
{
  source: "docs",
  clickId: "xxx"
}
```

Add `custom` parameters to parse more values from the url like `?utm_custom=123`
with `{ utm_custom: "custom" }` to get `{ custom: "123" }`.

## Invocations

To optimize the number of executions for e.g. performance optimizations or cost
reduction, these functions could help.

### debounce

Grouping multiple calls in a single invocation after a specific wait time.

```js
// debounce(fn: Function, wait=1000)
debounce(console.log)('called');
```

### throttle

Limit the number of invocations within a delay time.

```js
// throttle(fn: Function, delay=1000)
throttle(console.log)('called');
```

## isVisible

Used to check if an element is visible to the user. It follows the standard

```js
// isVisible(element: HTMLElement): boolean
isVisible(document.body) === true;
```
