# Session

Detect session client-side. There is no storage required, it's cookieless by
default.

Rules to start a new session:

1. (Optional) Read storage for an existing sessionId, if it's not existing, it's
   a `New Session`
2. If the `Entry Type` is a reload of a page, it's no new session
3. If there are any `Marketing parameters` in the URL, it's a new session
4. If the `Referrer` is different from the current domain, it's a new session

`Marketing Parameters` can be extended (default: utm_campaign, utm_content,
dclid, fbclid, gclid, utm_medium, msclkid, utm_source, utm_term)

`Referrer` domains can be extended, e.g. internal sub-domains. `Data` can be
pre-defined, e.g. to use your own id.

```js
startSession({
  data?: Walker.Properties;
  domains?: string[];
  isNew?: boolean;
  parameters?: MarketingParameters;
  referrer?: string;
  url?: string;
}) => Walker.Properties | false;
```
